面试官：你说一下 Agent Skill 的原理是什么？
  ▎
  ▎ 你：其实很简单。Skill 本质上就是给 LLM 提供的一系列「工具函数」和它们的定义说明。当 LLM 觉得需要调用某个工具时，它不
  ▎ 是用自然语言回复，而是输出一个结构化的「函数调用请求」。系统截获这个请求，执行对应的代码，把结果塞回给 LLM，LLM
  ▎ 再基于结果生成最终的回复。整个过程就像 LLM 在"指挥"外部工具帮它做事。

Skill 原理 = Function Calling + 函数注册 + 结果回填，让 LLM 从「只输出文本」变成「能调用工具」

LLM              │ 🧑‍💼 只会说话的前台接待                                                               │
  │ Skill            │ 📋 一张"工具说明书"，上面写着：按红色按钮→出咖啡，按蓝色按钮→出可乐                 │
  │ Function Calling │ 🧑‍💼 前台看完说明书，说"按红色按钮"，然后你去按                                       │
  │ MCP              │ 📏 规定"所有说明书都必须用A4纸、宋体、同一格式"，这样不管哪个前台都能看懂任何说明书
